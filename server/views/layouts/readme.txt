/*-----------------------------------------------------------------------------------------------*/
/*     layouts - шаблоны для сборки страниц                                                      */
/*-----------------------------------------------------------------------------------------------*/

• можно прописать layout "по умолчанию" через index.js. (сейчас отключен).
• перебить layout прописанный в index.js можно через параметр layout в контроллере соответствующей страницы: response.render('path', {layout: "layouts/<filename>"})
• в проекте, layouts подключены через файл server/config/layouts.js. Это сделано для создания распределенных интерфейсов. не стоит обходить этот механизм (!)

• layout [big_handbook] используется для справочников, имеющих собственную вкладку в хедере, таких как:
    - "заявки" ("/measurements.html")
    - "договоры" ("/contracts.html")
    - "отчеты" ("/reports.html")

• layout [main] - для таких страниц, как:
    - "справочники" (/handbooks.html)
    - "о программе"

• layout [management] - для страницы "админка" (/management.html)
• layout [login] - для страницы авториации (/login.html)
• layout [maintenance] - заглушка на период проведения ремонтных работ. Подключается по необходимости через server/config/variables.js

• справочники, загружаемые через сайдбар страниц "справочники" и "админка" не используют layouts, они загружаются через AJAX
